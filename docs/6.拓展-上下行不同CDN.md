# XHTTP + CDN 搭建指南 (六)：拓展 - 上下行不同 CDN

本文档用于在已有手动搭建配置基础上，增加一个新模式：

```text
上行 xhttp+TLS+CDN-A | 下行 xhttp+TLS+CDN-B
```

前提：你已经按前面的文档完成基础 XHTTP + CDN 配置，并且已有可用的 `xhttp+TLS 双向 CDN` 节点。

---

## 1. 工作方式

这个模式不需要新增 Xray 服务端入站。

它复用原来的：

```text
客户端 → CDN → VPS:443 → Xray 443 fallback → Nginx 8003 → grpc_pass 127.0.0.1:8001 → Xray XHTTP 入站
```

区别只是客户端连接方式：

- 上行连接到 `CDN-A`
- 下行 `download-settings` 连接到 `CDN-B`
- 两边使用相同的 `UUID / path / VLESS Encryption`

---

## 2. Cloudflare 设置

CDN-A 和 CDN-B 都需要：

- DNS 记录开启代理（橙色云朵）
- SSL/TLS 加密模式：完全（严格）
- Network 里开启 gRPC

如果 CDN-A 和 CDN-B 是同一个域名，只配置一次即可。

---

## 3. 证书

Nginx 需要能用同一张证书响应这些 CDN 域名。

假设：

```text
Reality 域名: reality.example.com
原 CDN 域名: cdn.example.com
CDN-A: cdn-a.example.com
CDN-B: cdn-b.example.com
```

重新签发证书时，把这些域名都带上：

```bash
systemctl stop nginx 2>/dev/null || true

acme.sh --issue \
  -d reality.example.com \
  -d cdn.example.com \
  -d cdn-a.example.com \
  -d cdn-b.example.com \
  --standalone \
  --keylength ec-256

acme.sh --install-cert -d reality.example.com --ecc \
  --key-file /etc/ssl/private/private.key \
  --fullchain-file /etc/ssl/private/fullchain.cer \
  --reloadcmd "systemctl restart nginx"
```

如果 VPS 使用 IPv6 申请证书，用下面这条完整命令：

```bash
systemctl stop nginx 2>/dev/null || true

acme.sh --issue \
  -d reality.example.com \
  -d cdn.example.com \
  -d cdn-a.example.com \
  -d cdn-b.example.com \
  --standalone \
  --listen-v6 \
  --keylength ec-256

acme.sh --install-cert -d reality.example.com --ecc \
  --key-file /etc/ssl/private/private.key \
  --fullchain-file /etc/ssl/private/fullchain.cer \
  --reloadcmd "systemctl restart nginx"
```

如果某些 CDN 域名相同，不要重复写 `-d`。

---

## 4. Nginx 配置

找到原 CDN 的 `server` 块，把 CDN-A / CDN-B 加到同一个 `server_name` 里。

原来类似：

```nginx
server {
    listen       8003 ssl;
    http2        on;
    server_name  cdn.example.com;

    ssl_certificate /etc/ssl/private/fullchain.cer;
    ssl_certificate_key /etc/ssl/private/private.key;

    location /your-xhttp-path {
        grpc_pass 127.0.0.1:8001;
        grpc_set_header Host                  $host;
        grpc_set_header X-Real-IP             $real_client_ip;
        grpc_set_header Forwarded             $proxy_add_forwarded;
        grpc_set_header X-Forwarded-For       $proxy_add_x_forwarded_for;
        grpc_set_header X-Forwarded-Proto     $scheme;
    }
}
```

改成：

```nginx
server {
    listen       8003 ssl;
    http2        on;
    server_name  cdn.example.com cdn-a.example.com cdn-b.example.com;

    ssl_certificate /etc/ssl/private/fullchain.cer;
    ssl_certificate_key /etc/ssl/private/private.key;

    location /your-xhttp-path {
        grpc_pass 127.0.0.1:8001;
        grpc_set_header Host                  $host;
        grpc_set_header X-Real-IP             $real_client_ip;
        grpc_set_header Forwarded             $proxy_add_forwarded;
        grpc_set_header X-Forwarded-For       $proxy_add_x_forwarded_for;
        grpc_set_header X-Forwarded-Proto     $scheme;
    }
}
```

如果 CDN-A / CDN-B 和原 CDN 有重复，只保留一次。

检查并重启：

```bash
nginx -t
systemctl restart nginx
```

## 5. Xray 服务端

Xray 服务端不用新增入站。

继续使用原来的 XHTTP 入站即可：

```json
{
  "listen": "127.0.0.1",
  "port": 8001,
  "protocol": "vless",
  "streamSettings": {
    "network": "xhttp",
    "xhttpSettings": {
      "path": "/your-xhttp-path",
      "mode": "auto"
    }
  }
}
```

如果你已经按 xpadding 文档给 XHTTP 入站加过 xpadding，这里保持原样即可。

## 6. Mihomo 客户端配置

基于原来的 `xhttp+TLS 双向 CDN` 节点复制一份，改成下面这样：

```yaml
  - name: 上行 xhttp+TLS+CDN-A | 下行 xhttp+TLS+CDN-B
    type: vless
    server: cdn-a.example.com
    port: 443
    uuid: YOUR_UUID2
    udp: true
    flow: ""
    tls: true
    network: xhttp
    alpn:
      - h2
    servername: cdn-a.example.com
    client-fingerprint: chrome
    encryption: YOUR_VLESS_ENCRYPTION
    xhttp-opts:
      host: cdn-a.example.com
      path: /your-xhttp-path
      mode: auto
      reuse-settings:
        max-concurrency: "16-32"
        c-max-reuse-times: "0"
        h-max-reusable-secs: "1800-3000"
      download-settings:
        host: cdn-b.example.com
        path: /your-xhttp-path
        server: cdn-b.example.com
        port: 443
        tls: true
        alpn:
          - h2
        servername: cdn-b.example.com
        client-fingerprint: chrome
        reality-opts: { public-key: "" }
        reuse-settings:
          max-concurrency: "16-32"
          c-max-reuse-times: "0"
          h-max-reusable-secs: "1800-3000"
```

如果原节点有 xpadding，就把这些字段也复制到 `xhttp-opts` 和 `download-settings` 里：

```yaml
      x-padding-obfs-mode: true
      x-padding-key: "x_padding"
      x-padding-header: "Referer"
      x-padding-placement: "queryInHeader"
      x-padding-method: "tokenish"
```

如果原节点有 ECH，就把 `ech-opts` 也复制到上行节点顶层和下行 `download-settings` 里。

## 7. v2rayN / Xray 客户端配置

客户端配置和双向CDN模式一样，核心是把上行地址改成 CDN-A，再在 `extra` 里加入 CDN-B 的 `downloadSettings`。

`downloadSettings` 示例：

```json
{
  "downloadSettings": {
    "address": "cdn-b.example.com",
    "port": 443,
    "network": "xhttp",
    "security": "tls",
    "tlsSettings": {
      "serverName": "cdn-b.example.com",
      "allowInsecure": false,
      "alpn": [
        "h2"
      ],
      "fingerprint": "chrome"
    },
    "xhttpSettings": {
      "host": "cdn-b.example.com",
      "path": "/your-xhttp-path",
      "mode": "auto"
    }
  }
}
```

如果使用 xpadding，把 xpadding 配置同步放进上行 `extra` 和下行 `downloadSettings.xhttpSettings.extra`。

如果使用 ECH，上行 TLS 和下行 `tlsSettings` 都要带上 ECH 配置。

---

## 8. 缓存绕过规则

Cloudflare Cache Rules 建议配置：

```text
(http.host eq "cdn-a.example.com") or (http.host eq "cdn-b.example.com") or (http.request.uri.path contains "/your-xhttp-path")
```

如果 CDN-A / CDN-B / 原 CDN 域名有重复，只保留一次。

如果想把原 CDN 也写进去，可以这样：

```text
(http.host eq "cdn.example.com") or (http.host eq "cdn-a.example.com") or (http.host eq "cdn-b.example.com") or (http.request.uri.path contains "/your-xhttp-path")
```

缓存资格设置为：

```text
绕过缓存
```
