# XHTTP + CDN 搭建指南 (七)：拓展 - 上下行 IPv4 / IPv6

本文档用于在已有手动搭建配置基础上，增加两个新模式：

```text
上行 xhttp+Reality IPv4 | 下行 xhttp+Reality IPv6
上行 xhttp+Reality IPv6 | 下行 xhttp+Reality IPv4
```

前提：你已经按前面的文档完成基础 XHTTP + CDN 配置，并且 VPS 的 IPv4 / IPv6 都能访问 443。

一键扩展脚本会自动获取 VPS IPv4 / IPv6 地址；手动搭建时可以用下面命令查看：

```bash
curl -4 -s ip.sb
curl -6 -s ip.sb
```

---

## 1. 工作方式

这个模式只走 Reality 直连，不走 CDN，也不需要 ECH。

需要准备两个 Reality 域名：

```text
IPv4 Reality 域名：reality-v4.example.com
IPv6 Reality 域名：reality-v6.example.com
```

客户端连接方式：

- 上行 IPv4 节点：连接 VPS IPv4，SNI 使用 `reality-v4.example.com`
- 下行 IPv6：`download-settings` 连接 VPS IPv6，SNI 使用 `reality-v6.example.com`
- 反向节点同理

---

## 2. DNS 设置

两个 Reality 域名都保持仅 DNS（灰色云朵）：

```text
reality-v4.example.com  A     VPS IPv4
reality-v6.example.com  AAAA  VPS IPv6
```

不要开启 CDN 代理。

---

## 3. 证书

Nginx 需要同一张证书响应原 Reality 域名、CDN 域名，以及新增的两个 Reality 域名。

示例：

```bash
systemctl stop nginx 2>/dev/null || true

acme.sh --issue \
  -d reality.example.com \
  -d cdn.example.com \
  -d reality-v4.example.com \
  -d reality-v6.example.com \
  --standalone \
  --listen-v6 \
  --keylength ec-256

acme.sh --install-cert -d reality.example.com --ecc \
  --key-file /etc/ssl/private/private.key \
  --fullchain-file /etc/ssl/private/fullchain.cer \
  --reloadcmd "systemctl restart nginx"
```

如果某些域名相同，不要重复写 `-d`。

---

## 4. Nginx 配置

给两个 Reality 域名各写一个独立回落站：

```nginx
server {
    listen       8003 ssl;
    http2        on;
    server_name  reality-v4.example.com;

    ssl_certificate /etc/ssl/private/fullchain.cer;
    ssl_certificate_key /etc/ssl/private/private.key;

    location / {
        proxy_pass https://www.stanford.edu;
        proxy_ssl_server_name on;
        proxy_ssl_name www.stanford.edu;
        proxy_set_header Host www.stanford.edu;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }
}

server {
    listen       8003 ssl;
    http2        on;
    server_name  reality-v6.example.com;

    ssl_certificate /etc/ssl/private/fullchain.cer;
    ssl_certificate_key /etc/ssl/private/private.key;

    location / {
        proxy_pass https://www.harvard.edu;
        proxy_ssl_server_name on;
        proxy_ssl_name www.harvard.edu;
        proxy_set_header Host www.harvard.edu;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }
}
```

检查：

```bash
nginx -t
systemctl restart nginx
```

## 5. Xray 服务端

Reality 入站需要监听 IPv4 / IPv6，并允许两个新的 SNI。

把 443 Reality 入站改成：

```json
{
  "listen": "::",
  "port": 443,
  "protocol": "vless",
  "streamSettings": {
    "network": "raw",
    "security": "reality",
    "realitySettings": {
      "target": "8003",
      "serverNames": [
        "reality.example.com",
        "reality-v4.example.com",
        "reality-v6.example.com"
      ],
      "privateKey": "YOUR_PRIVATE_KEY",
      "shortIds": [
        "YOUR_SHORT_ID"
      ]
    }
  }
}
```

原来的 XHTTP 入站 `127.0.0.1:8001` 不需要改。

检查并重启：

```bash
xray -test -config /usr/local/etc/xray/config.json
systemctl restart xray
```

## 6. Mihomo 客户端配置

基于原来的 `xhttp+Reality 上下行不分离` 节点复制两份。

### 上行 IPv4 / 下行 IPv6

```yaml
  - name: 上行 xhttp+Reality IPv4 | 下行 xhttp+Reality IPv6
    type: vless
    server: VPS_IPV4
    port: 443
    uuid: YOUR_UUID2
    udp: true
    flow: ""
    tls: true
    encryption: YOUR_VLESS_ENCRYPTION
    network: xhttp
    alpn:
      - h2
    servername: reality-v4.example.com
    client-fingerprint: chrome
    reality-opts:
      public-key: YOUR_PUBLIC_KEY
      short-id: YOUR_SHORT_ID
    xhttp-opts:
      path: /your-xhttp-path
      mode: auto
      reuse-settings:
        max-concurrency: "16-32"
        c-max-reuse-times: "0"
        h-max-reusable-secs: "1800-3000"
      download-settings:
        path: /your-xhttp-path
        server: "VPS_IPV6"
        port: 443
        tls: true
        alpn:
          - h2
        servername: reality-v6.example.com
        client-fingerprint: chrome
        reality-opts:
          public-key: YOUR_PUBLIC_KEY
          short-id: YOUR_SHORT_ID
        reuse-settings:
          max-concurrency: "16-32"
          c-max-reuse-times: "0"
          h-max-reusable-secs: "1800-3000"
```

### 上行 IPv6 / 下行 IPv4

把上面节点反过来：

```yaml
    server: "VPS_IPV6"
    servername: reality-v6.example.com
```

下行改成：

```yaml
        server: VPS_IPV4
        servername: reality-v4.example.com
```

如果原节点有 xpadding，把这些字段同时复制到 `xhttp-opts` 和 `download-settings`：

```yaml
      x-padding-obfs-mode: true
      x-padding-key: "x_padding"
      x-padding-header: "Referer"
      x-padding-placement: "queryInHeader"
      x-padding-method: "tokenish"
```

## 7. v2rayN / Xray 客户端配置

核心是：

- 上行地址填 VPS IPv4 / IPv6
- 上行 `sni` 填对应 Reality 域名
- `extra.downloadSettings.address` 填下行 IP
- `extra.downloadSettings.realitySettings.serverName` 填下行 Reality 域名

下行 `downloadSettings` 示例：

```json
{
  "downloadSettings": {
    "address": "VPS_IPV6",
    "port": 443,
    "network": "xhttp",
    "security": "reality",
    "realitySettings": {
      "show": false,
      "serverName": "reality-v6.example.com",
      "fingerprint": "chrome",
      "shortId": "YOUR_SHORT_ID",
      "publicKey": "YOUR_PUBLIC_KEY"
    },
    "xhttpSettings": {
      "host": "",
      "path": "/your-xhttp-path",
      "mode": "auto"
    }
  }
}
```

如果使用 xpadding，把 xpadding 配置同步放进上行 `extra` 和下行 `downloadSettings.xhttpSettings.extra`。
