# XHTTP + CDN 搭建指南 (一)：环境准备

本文档旨在记录并快速复现基于 `Xray-core` 的 xhttp 传输出站方案（支持 Reality 直连与 CDN 上下行分离）。

**参考资料**

1. 文档1：https://github.com/XTLS/Xray-core/discussions/4118
2. 文档2：https://jollyroger.top/sites/361.html

---

## 0. 准备工作 (材料清单)

在开始敲代码之前，请确保你已经准备好以下核心材料：

1. **一台 VPS**
2. **两个域名**：
   * `cdn.yourdomain2.com` -> 用于走 CDN 的流量
   * `reality.yourdomain1.com` -> 用于 Reality 直连伪装
3. **一个伪装路径**：可以随便写，例如 `/d3f2a994`

---

## 1. Cloudflare 设置

登录 Cloudflare 控制台，按顺序完成以下设置：

### 1.1 DNS 解析设置

* `cdn` 域名记录：**开启** 代理状态（点亮小黄云 橙色）。
* `reality` 域名记录：**关闭** 代理状态（仅限 DNS 灰色）。

### 1.2 SSL/TLS 加密模式

* 必须将加密模式设置为 **完全（严格） / Full (Strict)**。

### 1.3 开启 gRPC 转发

* 进入 **网络 (Network)**，向下滚动找到 **gRPC** 选项，并将其开关 **打开**。

### 1.4 配置缓存绕过规则（建议）

在 CF 配置缓存 bypass 规则，用于避免 XHTTP 请求被缓存。

面板左侧列表 → **缓存** → **Cache Rules** → **创建缓存规则**：

1. **名称**随意
2. 如果传入请求匹配... **选择** `自定义筛选表达式`
3. 往下滑，点击右侧蓝色的 **编辑表达式**
4. 输入以下内容（记得替换为你自己的域名和路径）：

```
(http.host eq "cdn.example.com") or (http.request.uri.path contains "/your-xhttp-path")
```

5. **缓存资格** 设置为 **绕过缓存**
6. 完成，点击 **部署** 即可

> 参考讨论：https://github.com/MetaCubeX/mihomo/discussions/2669#discussioncomment-16459799

## 3. 编译安装 Nginx 与服务配置

直接复制下方整段命令执行，完成 1.30.1 版本的编译安装：

```bash
# 先更新软件源索引
apt-get update

# 安装编译依赖并进行编译
apt-get install -y gcc g++ libpcre3 libpcre3-dev zlib1g zlib1g-dev openssl libssl-dev libcrypt-dev wget sudo make curl socat cron && wget https://nginx.org/download/nginx-1.30.1.tar.gz && tar -xvf nginx-1.30.1.tar.gz && cd nginx-1.30.1 && ./configure --prefix=/usr/local/nginx --sbin-path=/usr/sbin/nginx --conf-path=/etc/nginx/nginx.conf --with-http_stub_status_module --with-http_ssl_module --with-http_realip_module --with-http_sub_module --with-stream --with-stream_ssl_module --with-stream_ssl_preread_module --with-http_v2_module && make && make install && cd && mkdir /var/log/nginx
```

```bash
# 创建 Nginx 的 systemd 服务
cat > /etc/systemd/system/nginx.service << EOF
[Unit]
Description=A high performance web server and a reverse proxy server
Documentation=man:nginx(8)
After=network.target nss-lookup.target

[Service]
Type=forking
PIDFile=/run/nginx.pid
ExecStartPre=/usr/sbin/nginx -t -q -g 'daemon on; master_process on;'
ExecStart=/usr/sbin/nginx -g 'daemon on; master_process on;'
ExecReload=/usr/sbin/nginx -g 'daemon on; master_process on;' -s reload
ExecStop=-/sbin/start-stop-daemon --quiet --stop --retry QUIT/5 --pidfile /run/nginx.pid
TimeoutStopSec=5
KillMode=mixed

[Install]
WantedBy=multi-user.target
EOF
```

```bash
# 重载并设置开机启动
systemctl daemon-reload && systemctl enable nginx.service
```

## 4. 安装 Acme.sh 并正式部署双域名证书

```bash
# 安装 acme 环境
apt install socat cron -y && curl https://get.acme.sh | sh && ln -s /root/.acme.sh/acme.sh /usr/local/bin/acme.sh && ln -s /root/.acme.sh/acme.sh /usr/local/bin/acme.sh

# 切换 CA 为 Let's Encrypt
acme.sh --set-default-ca --server letsencrypt

# 申请双域名 ECC 证书
acme.sh --issue -d reality.yourdomain1.com -d cdn.yourdomain2.com --standalone --keylength ec-256 \
  --pre-hook "systemctl stop nginx 2>/dev/null || true" \
  --post-hook "systemctl start nginx 2>/dev/null || true"

# 创建证书存放目录
mkdir -p /etc/ssl/private

# 将证书正式安装到系统目录
acme.sh --install-cert -d reality.yourdomain1.com --ecc \
  --key-file /etc/ssl/private/private.key \
  --fullchain-file /etc/ssl/private/fullchain.cer \
  --reloadcmd "systemctl restart nginx"
```
